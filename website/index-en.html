Core features

Not a generic RPA demo—tuned for real Office workflows.

🎬 One-time recording

Capture clicks, typing, hotkeys, scroll, and drag-select. Merges consecutive keys to keep flows lean.

🧱 Reusable modules

Save “copy → open web → paste” as a block and insert it anywhere—no re-recording.

🔁 Table loops

Drag-select rows in Excel or enter row count in WPS; record the loop body once, repeat N times.

🌐 Browser-aware

Tuned for Gmail and internal web apps—paste timing and load waits to reduce focus mistakes.

🖥️ Web console

Optional local web UI to manage flows; tunnel for remote trigger (execution still on the office PC).

🔒 Local execution

No cloud bot holding your passwords. Flows stored as JSON on disk—good for locked-down networks.

FAQ

Is this SaaS? Is my data uploaded?

No. Flows and optional screenshots stay in your local user folder. The web console is optional and does not require a public cloud.

Can it control WPS or WeChat on my PC?

It drives normal desktop UI (clicks and keyboard) in installed apps. Some security tools or remote-desktop layers may block simulated input.

Same features on Mac and Windows?

Core record/playback is the same. Excel drag-to-loop is strongest on macOS; on Windows, row-count input is often easier.

Why should investors care?

Long-tail automation for SMBs and teams—lighter than enterprise RPA, easier than scripts; modules create reuse and expansion.
